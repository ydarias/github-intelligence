# Reporter

Another application will be a UI to display the issues, and create multiple reports and stats.

The UI will be implemented using the [React](https://reactjs.org/) library. But I am no expert so you must choose simplicity and readability over smart solutions.

The app will run locally, and will read the issues from the repository at issues-collector.

Besides running locally it should use pagination, displaying 50 issues per page by default.
